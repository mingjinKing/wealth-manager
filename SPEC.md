# 消费透视镜 (Consumer Telescope) - 项目规格说明书

> V1 聚焦版 —— 纯工具定位
> 版本：v1.2 | 更新：2026-03-31 | 变更：哇时刻公式明确 + AI方案确定 + 颜色系统统一

---

## 1. 项目概述

**项目名称**：消费透视镜
**项目类型**：Android 原生应用
**一句话描述**：像钱迹一样方便快捷的记账工具，加上 AI 消费结构透视能力。

---

## 2. 核心功能

- **快速记账**：3秒内完成一笔记账，极简流程
- **消费结构透视**：AI 优化建议 + 哇时刻成就感
- **钱迹数据导入**：CSV 格式导入，历史数据无缝衔接
- **本地优先存储**：SQLite + 文件备份，隐私第一

---

## 3. 技术架构

| 层级 | 技术选型 |
|------|----------|
| **前端** | Android Native（Kotlin + Jetpack Compose） |
| **本地数据库** | SQLite（通过 Room 抽象） |
| **AI 能力** | LLM API（MiniMax/DeepSeek/阿里百炼），V1 使用已有算力资源 |
| **网络** | Retrofit + OkHttp |
| **依赖注入** | Hilt |
| **异步处理** | Kotlin Coroutines + Flow |
| **架构模式** | MVVM + 简化 Clean Architecture（2层：data + presentation）|
| **后台任务** | WorkManager（哇时刻定时计算） |

## 4. V1 MVP 功能列表

### 4.1 消费记录模块
- 添加消费记录（金额、分类、日期、备注）
- 消费分类管理（餐饮、购物、交通、娱乐等默认分类）
- 消费记录列表展示与搜索
- 编辑/删除消费记录

### 4.2 数据存储模块
- Room 数据库设计与实现
- 消费记录表、分类表
- 本地数据持久化

### 4.3 钱迹导入模块
- CSV 格式解析
- 历史数据导入与去重
- 分类映射（钱迹分类 → 本应用分类）

### 4.4 AI 透视模块（V1）
- **优化建议**：LLM 生成具体可执行的省钱建议（如"外卖每周减一次，月省 ¥200"）
- 通过 LLM API 调用实现（使用已有算力资源）
- 消费对比、趋势分析：后续版本支持

### 4.5 第一个"哇"时刻 ⭐（v1.2 核心变更）

**定义**：每周日向用户展示"因为消费透视镜，你本周少花了 ¥XX"。

**触发公式（v1.2 明确版）**：
```
savedAmount = last_4_week_avg - current_week_spend
wow_triggered = savedAmount > ¥100 AND savedAmount > last_4_week_avg × 20%
```

**详细说明**：
- `last_4_week_avg`：用户历史最近4周消费均值（滚动计算）
- `current_week_spend`：本周消费总额
- `savedAmount`：节省金额（正数表示少花了）
- 触发条件：**绝对节省 > ¥100** 且 **相对节省 > 历史均值20%**（两者同时满足）
- 不满足条件时不展示哇时刻，不打扰用户
- 新用户（数据不足4周）：使用可配置的默认基准值

**哇时刻展示内容**：
- 本周节省金额
- 原因描述（"因为你少点了N次外卖，节省了 ¥XX"）
- 等价物展示（"相当于：一顿双人火锅"）
- 成就徽章

### 4.6 基础 UI（v1.2 更新）

- **首页仪表盘**：消费结构 + AI建议 + 哇时刻预览 + BottomNav
- **记账页**：3秒极速录入
- **透视页**：AI 建议详情
- **成就页**：哇时刻 + 成就徽章
- **导入页**（钱迹 CSV）
- **设置页**（AI API 配置）

**BottomNav 4栏**：
| 图标 | 标签 | 页面 |
|------|------|------|
| 🏠 | 首页 | 仪表盘 |
| ➕ | 添加 | 记账页 |
| 💡 | 透视 | AI 建议页 |
| 🏆 | 成就 | 哇时刻 + 成就列表 |

## 5. 非功能需求

- 最低支持 Android 7.0（API 24）
- 离线优先，数据本地存储
- 用户隐私数据不上传第三方服务器
