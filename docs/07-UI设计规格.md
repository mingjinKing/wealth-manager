### 4.4 月度概览卡片（MonthOverviewCard）

- **比例**：4:3（aspectRatio = 4f / 3f）
- **背景**：可切换的背景图片（drawable 目录下的 `background_1`、`background_2`）
- **点击交互**：点击卡片区域切换背景图片
- **渐变蒙层**：黑色透明渐变（0.4f → 0 → 0.3f），确保文字在亮色背景上清晰
- **数据展示**：
  - 本月支出：大号金色数字（42sp Bold）
  - 本月收入：白色中号文字
  - 近7日支出：白色中号文字

### 4.5 每日账单卡片交互

- **点击单条消费条目** → 进入记账页编辑
- **长按单条消费条目** → 弹出删除确认对话框
- **编辑流程**：点击 → 修改金额/分类/备注/日期 → 确认 → 返回首页更新
- **删除流程**：长按 → 确认弹窗 → 确认删除 → 返回首页更新

### 4.6 空状态

当无记账记录时，显示：
- 主文案："暂无记账记录"
- 副文案："点击 + 开始记账"

## 五、记账页 · AddExpenseScreen

### 5.1 屏幕类型

**全屏沉浸式数字输入页**。

### 5.2 核心组件

| 组件名称 | 行为 |
|----------|------|
| 顶部导航 | 返回箭头 + 标题"添加账单"/"修改账单" + 日期选择器 |
| Tab 切换 | "添加账单" / "添加收入" 两个 Tab |
| 分类选择器 | 4列网格布局，emoji + 文字，选中态实心金色填充 |
| 音符输入 | BasicTextField，最多50字，与金额并排一行 |
| 金额显示 | 实时显示，数字键盘输入，支持 +/- 计算 |
| 数字键盘 | 5列布局：数字 + 功能键 |
| 日期选择器 | 右上角可点击，弹出 DatePickerDialog |

### 5.3 布局结构

```
┌──────────────────────────────────────┐
│ [←]    添加账单              [今天] │  ← 可点击选择日期
│ ─────────────────────────────────── │
│  [添加账单]        [添加收入]        │  ← Tab 切换
│ ─────────────────────────────────── │
│                                      │
│  [🍗餐饮] [🛒购物] [🚇交通] [🎮娱乐]│  ← 分类网格（4列）
│  [🏠居住] [💊医疗] [📚学习] [📦其他]│
│                                      │
│  ┌────────────────┐                  │
│  │ 备注            │    ¥ 0        │  ← 音符+金额并排
│  └────────────────┘                  │
│                                      │
│   1   2   3   +   −                │
│   4   5   6   ⌫   ✓                │  ← 数字键盘（5列）
│   7   8   9   ·                   │
│   0       0                   │
└──────────────────────────────────────┘
```

### 5.4 数字键盘布局

| 行 | 按键 |
|----|------|
| 1 | `7` `8` `9` `+` `−` |
| 2 | `4` `5` `6` `⌫` `✓` |
| 3 | `1` `2` `3` `·` |
| 4 | `0` (占2列宽度) |

### 5.5 计算器逻辑

- 支持 + / − 运算
- 输入数字 → 点击运算符 → 输入第二个数字 → 点击 ✓ 计算结果
- 显示实时预览（pendingValue + pendingOperator）

### 5.6 日期选择

- 默认显示"今天"，点击弹出 Material3 DatePickerDialog
- 日期显示格式：今天 / 昨天 / 4月2日
- 选定日期随账单一起存入数据库

### 5.7 分类长按删除

- 长按分类 → 弹出删除确认对话框
- 删除后自动取消选中状态

### 5.8 记账完成流程

1. 选择分类 → 输入金额 → ✓ 确认
2. 新建：自动跳转回首页（Dashboard）
3. 编辑：返回上一级（popBackStack）

6. 解释存入 AgentContext.user_memories，下次分析携带记忆

## 十一、哇时刻触发公式

```
savedAmount = last_4_week_avg - current_week_spend
wow_triggered = savedAmount > ¥100 AND savedAmount > last_4_week_avg × 20%
```

| 参数 | 定义 |
|------|------|

| 兜底 | 不足4周数据时取默认值 |

## 十二、数据模型（Room 5张表，以代码为准）

> ⚠️ 以下数据模型以**代码实现**为准。文档曾有不一致，已修正。

### 12.1 CategoryEntity（分类表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键，自增 |
| name | String | 分类名称 |
| icon | String | emoji 图标 |
| color | String | 颜色（十六进制，如 `#ffc880`）|
| type | String | 类型：`"EXPENSE"`（支出）或 `"INCOME"`（收入），默认 `"EXPENSE"` |
| isDefault | Boolean | 是否为默认分类，默认 `true` |

**默认支出分类**（8个）：🍗 餐饮、🛒 购物、🚇 交通、🎮 娱乐、🏠 居住、💊 医疗、📚 学习、📦 其他
**默认收入分类**（8个）：💰 工资、🧧 奖金、🚲 兼职、📈 理财、🎁 礼金、📝 报销、♻️ 转卖、💸 其他收入

> **收入/支出区分方式**：不通过 `ExpenseEntity.type` 字段区分，而是通过关联的 `CategoryEntity.type` 属性区分。

### 12.2 ExpenseEntity（消费记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键，自增 |
| amount | Double | 金额（**正数**，支出/收入由 Category.type 区分）|
| categoryId | Long | 分类ID（外键，指向 CategoryEntity）|
| note | String | 备注（最多50字）|
| date | Long | 日期时间戳 |
| createdAt | Long | 创建时间，默认 `System.currentTimeMillis()` |

> ⚠️ **注意**：`ExpenseEntity` **没有** `type` 字段。收入/支出通过 `CategoryEntity.type`（"EXPENSE"/"INCOME"）区分。

### 12.3 WeekStatsEntity（周统计表）

| 字段 | 类型 | 说明 |
|------|------|------|
| weekStartDate | Long | **主键**，周开始日期时间戳 |
| totalAmount | Double | 周消费总额（**不含收入**）|
| categoryBreakdown | String | 分类消费明细（JSON 格式）|
| wowTriggered | Boolean | 是否触发哇时刻 |
| savedAmount | Double | 本周节省金额（正数=少花了）|

### 12.4 AssetEntity（资产表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键，自增 |
| name | String | 资产名称 |
| type | AssetType | 资产类型枚举（CASH / BANK / ALIPAY / INVESTMENT / CREDIT_CARD / LOAN / DEPOSIT / HOUSING_FUND / ENTERPRISE_ANNUITY / OTHER / CUSTOM）|
| customType | String? | 自定义账户类型名称（当 type=CUSTOM 时）|
| balance | Double | 当前余额 |
| icon | String | emoji 图标，默认 `"💰"` |
| color | String | 颜色（十六进制），默认 `"#4CAF50"` |
| isHidden | Boolean | 是否不计入总资产，默认 `false` |
| createdAt | Long | 创建时间，默认 `System.currentTimeMillis()` |

### 12.5 BudgetEntity（预算表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键，自增 |
| amount | Double | 预算金额 |
| month | String | 预算月份（格式：`"yyyy-MM"`，如 `"2026-04"`）|
| categoryId | Long? | 分类ID（**可为空**，`null` = 全局预算）|
