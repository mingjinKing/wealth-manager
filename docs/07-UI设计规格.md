## 二、设计决策总览（v2.0 重大变更）

### 主题切换：暗色 → 浅色

| 区域 | 旧（v1.2）| 新（v2.0）|
|------|------------|------------|
| 全局背景 | `#0c0c1f` 深墨色 | `#F0F0F0` 浅灰色 |
| 页面卡片 | `#13132b` | `#FFFFFF` 纯白 |
| 底部导航 | `#13132b` | `#FFFFFF` 纯白 |
| 文字 | 白色 | 深色 `#1A1A2E` |

### 首页重构：周概览 → 日账单

| | 旧（v1.2）| 新（v2.0）|
|--|------------|------------|
| 概览 | 本周总计 + 环比变化 | 本月支出 + 近7日支出 |
| 明细 | 消费结构进度条（按分类）| 每日收支分组（今天/昨天/M月d日）|
| AI建议 | 在首页（已移除）| 归入透视页 |

### 记账页重构

| 改动 | 详情 |
|------|------|
| 分类带图标 | 每个分类 Chip 显示 emoji + 名称（如 🍗 餐饮）|
| 备注支持 | 音符与金额并排一行，最多50字 |
| 日期选择 | 右上角可点击，弹出 DatePicker 选择具体日期 |
| 数字键盘 | 5列布局，+/- 替换部分功能键 |

### 4.4 每日账单卡片交互

- 点击单条消费条目 → 底部弹出编辑/删除菜单（ModalBottomSheet）

- 选择「删除」→ 确认对话框

### 4.5 首页不包含的内容

> AI 优化建议已从首页移除，统一归入「透视」页面。

## 五、记账页 · 3秒极速录入（Quick Add）

### 5.1 屏幕类型
**全屏沉浸式数字输入页**。

### 5.2 核心组件

| 组件名称 | 行为 |
|----------|------|
| 顶部导航 | 返回箭头 + 标题"添加账单" + 日期选择器 |
| 分类选择器 | FlowRow 排列，emoji + 文字，选中态实心金色填充 |
| 音符输入 | BasicTextField，最多50字，与金额并排一行 |
| 金额显示 | 实时显示，数字键盘输入 |
| 数字键盘 | 5列布局：1-9 + +/− + ⌫ / 4-6 + ✓ + 0 |
| 日期选择器 | 右上角可点击，弹出 Material3 DatePicker |

### 5.3 布局结构

```
┌──────────────────────────────────────┐
│ [←]        添加账单          [今天] │  ← 可点击选择日期
│                                      │
│  [🍗餐饮] [🛒购物] [🚇交通] [🎮娱乐]│  ← 分类 Chips
│  [🏠居住] [💊医疗] [📚学习] [📦其他]│
│                                      │
│  ┌────────────────┐                  │
│  │ 备注            │    ¥ 0        │  ← 音符+金额并排
│  └────────────────┘                  │
│                                      │
│   1   2   3   +   −                │
│   4   5   6   ⌫   ✓                │  ← 数字键盘（5列）
│   7   8   9   ·   00               │
│   0                               │
└──────────────────────────────────────┘
```

### 5.4 日期选择

- 默认显示"今天"，点击弹出 Material3 DatePickerDialog
- 日期显示格式：今天 / 昨天 / 3月31日
- 选定日期随账单一起存入数据库

### 5.5 记账完成流程

1. 选择分类 → 输入金额 → ✓ 确认
2. 新建：自动跳转回首页（Dashboard）
3. 编辑：返回上一级（popBackStack）

## 九、页面流转（v2.0）

```

    ↓
记账页（添加账单）
    ↓ [✓ 确认]
自动跳转 → 首页（消费仪表盘）
    ↓ [FAB +]
记账页（新建模式）
    ↓ [点击消费条目]
底部弹窗（修改/删除）
    ↓ [修改]
记账页（编辑模式，预填数据）
    ↓ [✓ 确认]
返回首页（更新数据）
    ↓ [BottomNav 透视]
AI 透视页（建议待LLM接入）
    ↓ [BottomNav 成就]
成就页（哇时刻 + 成就列表）
    ↓ [查看透视 →]
AI 透视页
    ↓
每周结算触发（每周日 20:00，WorkManager）
    ↓
哇时刻（成就页内卡片触发）
```

## 十、哇时刻触发公式

```
savedAmount = last_4_week_avg - current_week_spend
wow_triggered = savedAmount > ¥100 AND savedAmount > last_4_week_avg × 20%
```

| 参数 | 定义 |
|------|------|
| `last_4_week_avg` | 最近4周消费均值（滚动计算）|
| `current_week_spend` | 本周消费总额 |
| `savedAmount` | 节省金额（正数=少花了）|
| 兜底 | 不足4周数据时取默认值 |
