## 一、核心定位

> **知其财，治其财** —— 了解你的财富，管理你的财富

**一句话定位**：像钱迹一样方便快捷的记账工具，加上 AI 消费结构透视能力。

**核心价值主张**

- **透视感**：一眼看出"钱花哪去了"，不是数字是洞察
- **对照感**：有参照系，知道自己"偏没偏"
- **行动感**：不只是展示，是告诉你"可以怎么改"
- **治理感**：管理你的资产，掌握你的财务健康

## 一.5 产品哲学

### 核心理念

> **极简操作，静默守护，适时出现。**

| 原则 | 说明 |
|------|------|
| **极简操作** | 核心记账业务流程：打开 App → 记账 → 退出。全程无任何干扰，无弹窗，无打断。 |
| **静默守护** | 后台默默分析、处理、记录。不出声，不刷存在感。 |
| **适时出现** | 只在用户**主动**复盘时，才站出来给总结/建议。用户没问，绝不主动说。 |
| **不离不弃** | 即使用户最终没采纳建议，也没关系。依然在，依然守护。 |

### 产品人格：旺财

**旺财**是知财的 AI 管家，负责：
- 静默分析：记账数据进来，默默计算、分类、挖掘
- 总结汇报：用户主动来看周报/月报时，呈现洞察
- 建议生成：用户问"我这周花了多少"时，给出建议
- 永远在线：用户采纳也好，没采纳也好，旺财都在

**旺财的背后**：
- 规则引擎（精准计算）
- LLM（语义理解、洞察生成）
- 行为经济学、心理学、认知心理学（让建议真正有用、让人愿意听）

### 使命

> **帮助普通老百姓精打细算，花该花的，不花冤枉钱。**

财富增长的第一杠杆不是「投资收益」，而是「少花冤枉钱」。

## 三、资产管理

### 3.1 功能描述

管理用户的资产账户，包括：
- 现金
- 银行卡
- 支付宝
- 微信
- 投资账户（基金、股票等）

### 3.2 数据模型

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键 |
| name | String | 资产名称（如"招商银行储蓄卡"）|
| amount | Double | 当前金额 |
| type | String | 资产类型 |

### 3.3 用户价值

- 汇总查看所有资产的总和
- 按类型分类管理
- 了解资金分布

### 4.2 数据模型

- 提前预警，避免月光

## 五、成就体系

### 5.1 第一个"哇"时刻 ⭐

> **"攒到的就是赚到的"**

产品帮助用户攒下多少钱——这是给予用户的巨大正面反馈。

#### 触发公式（明确版）

```
savedAmount = last_4_week_avg - current_week_spend
wow_triggered = savedAmount > ¥100 AND savedAmount > last_4_week_avg × 20%
```

#### 详细说明

| 参数 | 定义 |
|------|------|
| `last_4_week_avg` | 用户历史最近4周消费均值（滚动计算，不足4周用默认值）|
| `current_week_spend` | 本周消费总额 |
| `savedAmount` | 节省金额（正数 = 少花了）|

#### 触发条件（两者同时满足）

1. **绝对节省 > ¥100** — 避免 trivial win（少于 ¥100 不打扰用户）
2. **相对节省 > 历史均值 × 20%** — 确保有意义的进步

#### 兜底规则

- 新用户（数据不足4周）：使用用户设定的周预算基准，或系统默认值
- 异常高消费周（如医疗紧急）：基准自适应，排除异常周
- 每周结算时间：**周日 20:00**，自动计算并推送通知

#### 哇时刻展示内容

| 元素 | 说明 |
|------|------|
| 节省金额 | 金色超大字号 ¥XX |
| 原因描述 | "因为你少点了N次外卖，节省了 ¥XX" |
| 等价物 | "相当于：一顿双人火锅" |
| 成就徽章 | 🏆 精打细算 等 |
| 进度暗示 | "连续4周达标解锁 → 攒钱达人" |

#### 不触发的情况

- 节省金额 ≤ ¥100 → 不展示
- 节省金额 ≤ 历史均值 × 20% → 不展示
- 消费 ≥ 上周（未节省）→ 不展示，沉默处理
