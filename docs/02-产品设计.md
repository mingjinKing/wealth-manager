## 一、核心功能（以代码为准）

### 1.1 消费记录模块

- ✅ 添加消费/收入记录（金额、分类、日期、备注）
- ✅ 消费分类管理（默认分类带 emoji 图标：餐饮/购物/交通/娱乐/居住/医疗/学习/其他 + 收入分类）
- ✅ 消费记录按天分组展示（今日/昨天/M月d日）
- ✅ 点击条目 → 进入编辑页
- ✅ 长按条目 → 删除确认对话框
- ✅ 编辑消费记录（预填数据，保存为更新）
- ✅ 收入/支出通过 CategoryEntity.type 区分（EXPENSE / INCOME）

### 1.2 资产管理模块

- ✅ 查看资产列表
- ✅ 添加/编辑/删除资产账户
- ✅ 按 AssetType 枚举分类（CASH / BANK / ALIPAY / INVESTMENT / CREDIT_CARD / LOAN / DEPOSIT / HOUSING_FUND / ENTERPRISE_ANNUITY / OTHER / CUSTOM）
- ✅ 支持自定义类型名称（customType）
- ✅ 支持隐藏账户不计入总资产（isHidden）
- ✅ 计算净资产（总资产 + 负债）

### 1.3 预算管理模块

- ✅ 按月设置分类预算
- ✅ 全局预算（categoryId = null）和分类预算并存
- ✅ 月度/周度预算二选一（根据是否有值自动切换）
- ✅ 追踪预算执行进度

### 1.4 周统计模块

- ✅ WeekStatsEntity 滚动计算
- ✅ 近4周均值计算（用于哇时刻）
- ✅ categoryBreakdown JSON 格式存储分类明细

### 1.5 钱迹数据导入

- ✅ 支持钱迹 CSV 格式导入（18列）
- ✅ 只导入支出记录
- ✅ 自动分类匹配

### 1.6 成就体系

- ✅ 哇时刻预览卡片
- ✅ 成就列表展示
- ✅ 净资产追踪
- ✅ 目标进度展示（TrendPoint）

### 1.7 透视页（✅ 已接入 LLM）

- ✅ AI 复盘按钮（"AI 复盘"），位于 TopAppBar右侧
- ✅ 旺财 Micro Agent v1.9.1 + DeepSeek-v3.2（流式输出）
- ✅ 实时进度提示（"正在梳理指定范围内的账单..."、"正在为您撰写专属财务报告..."）
- ✅ 用户解释反馈（用户可对 AI 分析进行解释/反驳，记忆存入 AgentContext）
- ✅ 本地规则兜底（规模/结构/频率分析，LLM 不可用时依然生效）

### 1.8 怎么花（规划中）

> 状态：待开发 v2.5
> 定位：帮助用户花好每一笔钱的决策助手

**核心价值：** 不是"不该花"，而是"怎么花才值" — 第三方顾问角色

**产品形态：**
- 底部导航栏独立 Tab（4栏：首页/算算账/怎么花/进阶）
- 问答式对话交互
- 预设引导流程，帮助用户开口

**用户场景：**
```
用户：我想买一台相机，预算 1 万
产品：结合你的消费习惯、历史支出、公开价格信息...
      → 建议：可以买，但建议等 618 或看看二手
      → 理由：1万占月收入30%、历史价格在9月更低
```

**信息输入：**
```
├── 用户画像
│   ├── 收入水平、月均消费
│   ├── 历史消费习惯（冲动型 vs 理性型）
│   ├── 储蓄目标
│   └── 已有设备/类似消费记录
│
├── 公开信息
│   ├── 历史价格走势
│   ├── 近期促销预测
│   ├── 竞品对比
│   └── 社交媒体评价（舆情）
│
└── 个人场景
    ├── 紧急程度
    ├── 替代方案（二手/租/平替）
    └── 购买时机
```

**引导流程设计：**
| 步骤 | 引导问题 |
|------|----------|
| 1 | 🎯 "你想买什么？买东西、做投资，还是日常开销？" |
| 2 | 💰 "你的预算是多少？" |
| 3 | 📝 "这个对你来说紧急还是可替代的？" |
| 4 | 📊 结合消费习惯 + 公开信息 → 生成建议 |

**与透视页的关系：**
```
透视页：被动分析（你已经花了，我来分析）
怎么花：主动决策（你要花，我帮你判断）
  ↓               ↓
知道过去       决策未来
```

**技术需求：**
| 能力 | 说明 |
|------|------|
| 对话式 UI | 类似透视页的 AI 交互 |
| 引导流程 | 预设问题引导用户 |
| 用户画像 | 结合历史消费数据 |
| 外部信息 | 价格/API/搜索/舆情 |
| LLM | 生成个性化建议 |

```kotlin
val avgLast4Weeks = recentStats.take(4).map { it.totalAmount }.average()
val savedAmount = avgLast4Weeks - recent7DaysTotalExpense

```

## 六、本地记忆系统

> 详见 `docs/03-技术架构.md` 第八章。

### 6.1 设计理念

- **本地化优先**：所有记忆存储在本地 SQLite，保护用户隐私
- **轻量化检索**：FTS5/FTS4 全文搜索 + 向量相似度搜索 + RRF 融合
- **AI 自动提炼**：通过钩子触发，AI 从对话中自动提取关键事实
- **用户可控**：记忆可管理、可删除、可纠错

### 6.2 三层记忆架构

| 层级 | 内容 | 存储位置 | 生命周期 |
|------|------|----------|----------|
| **工作记忆** | 当前 session 的完整上下文 | 内存 (StateFlow) | session 结束即清除 |
| **短期记忆** | 提炼自对话的结构化事实 | `extracted_facts` 表 | 30 天无召回则过期 |
| **长期记忆** | 多次验证一致的重要信息 | `memory` 表 | 90 天或被明确更新 |

### 6.3 关键信息提取流程

```
对话 → Hook 触发 → AI 提取关键事实 → FTS4 存储 + 向量嵌入 → 检索使用
```

**触发时机**：AI 回复完成后自动触发（post-hook），异步执行不阻塞主流程

**提取内容**：
- 用户透露的事实（预算、偏好、目标、限制条件）
- 对话中产生的决定/结论/建议

**存储格式**：
```
ExtractedFactEntity
    ├── key      → 语义化键（budget_clothes_monthly）
    ├── value    → JSON 结构化值
    ├── summary  → 人类可读摘要（向量嵌入对象）
    └── confidence → 置信度 0-1
```

### 6.4 检索策略

- **RRF 融合**：FTS4 关键词检索 + 向量相似度搜索，融合排名
- **时间衰减**：记忆相关性随时间衰减
- **置信度过滤**：低于阈值的事实不注入上下文
