# 知财 · 技术架构

> 整理：玄枢·麒麟 × 乘黄 | 最后更新：2026-04-02 | 版本：v2.3

## 一、技术架构

| 模块 | 方案 | 理由 |
|------|------|------|
| 记账入口 | 原生开发（安卓优先）| 本地性能+隐私控制 |
| 数据存储 | SQLite（本地）+ Room 抽象 | 隐私优先，钱迹同款方案 |
| 账单分析 | 本地AI推理（小模型）/ 外部 API | 不强依赖网络 |
| 数据来源 | V1：手动记账 | 降低门槛，合规 |
| UI 框架 | Jetpack Compose + Material3 | 现代声明式 UI |

### 技术栈总览

| 层级 | 技术选型 |
|------|----------|
| **前端** | Android Native（Kotlin + Jetpack Compose） |
| **本地数据库** | SQLite（通过 Room 抽象） |
| **AI 能力** | LLM API（MiniMax/DeepSeek/阿里百炼）|
| **网络** | Retrofit + OkHttp |
| **依赖注入** | Hilt |
| **异步处理** | Kotlin Coroutines + Flow |
| **架构模式** | MVVM + 简化 Clean Architecture（2层：data + presentation）|
| **后台任务** | WorkManager（哇时刻定时计算）|

## 二、Room 数据库架构（5张表）

> ⚠️ 以下以**代码实现**为准，文档曾有不一致，已修正。

### 2.1 数据库概览

```
AppDatabase (wealth_manager_db, version 4)
├── CategoryDao
│   └── CategoryEntity (分类表)
├── ExpenseDao
│   └── ExpenseEntity (消费记录表)
├── WeekStatsDao
│   └── WeekStatsEntity (周统计表)
├── AssetDao
│   └── AssetEntity (资产表)
└── BudgetDao
    └── BudgetEntity (预算表)
```

### 2.2 CategoryEntity（分类表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键，自增 |
| name | String | 分类名称 |
| icon | String | emoji 图标 |
| color | String | 颜色（十六进制，如 `#ffc880`）|
| type | String | 类型：`"EXPENSE"`（支出）或 `"INCOME"`（收入），默认 `"EXPENSE"` |
| isDefault | Boolean | 是否为默认分类，默认 `true` |

**默认支出分类**（8个）：
- 🍗 餐饮、🛒 购物、🚇 交通、🎮 娱乐、🏠 居住、💊 医疗、📚 学习、📦 其他

**默认收入分类**（8个）：
- 💰 工资、🧧 奖金、🚲 兼职、📈 理财、🎁 礼金、📝 报销、♻️ 转卖、💸 其他收入

> **收入/支出区分方式**：不通过 `ExpenseEntity.type` 字段区分，而是通过关联的 `CategoryEntity.type` 属性区分。

### 2.3 ExpenseEntity（消费记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键，自增 |
| amount | Double | 金额（**正数**，支出/收入由 Category.type 区分）|
| categoryId | Long | 分类ID（外键，指向 CategoryEntity）|
| note | String | 备注（最多50字）|
| date | Long | 日期时间戳 |
| createdAt | Long | 创建时间，默认 `System.currentTimeMillis()` |

### 2.4 WeekStatsEntity（周统计表）

| 字段 | 类型 | 说明 |
|------|------|------|
| weekStartDate | Long | **主键**，周开始日期时间戳 |
| totalAmount | Double | 周消费总额（**不含收入**）|
| categoryBreakdown | String | 分类消费明细（JSON 格式）|
| wowTriggered | Boolean | 是否触发哇时刻 |
| savedAmount | Double | 本周节省金额（正数=少花了）|

### 2.5 AssetEntity（资产表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键，自增 |
| name | String | 资产名称 |
| type | AssetType | 资产类型枚举（见下）|
| customType | String? | 自定义账户类型名称（当 type=CUSTOM 时）|
| balance | Double | 当前余额 |
| icon | String | emoji 图标，默认 `"💰"` |
| color | String | 颜色（十六进制），默认 `"#4CAF50"` |
| isHidden | Boolean | 是否不计入总资产，默认 `false` |
| createdAt | Long | 创建时间，默认 `System.currentTimeMillis()` |

**AssetType 枚举**：

| 枚举值 | 说明 |
|--------|------|
| CASH | 现金 |
| BANK | 银行卡 |
| ALIPAY | 支付宝/微信 |
| INVESTMENT | 投资账户 |
| CREDIT_CARD | 信用卡（负债）|
| LOAN | 借入/借出 |
| DEPOSIT | 存款 |
| HOUSING_FUND | 公积金 |
| ENTERPRISE_ANNUITY | 企业年金 |
| OTHER | 其他 |
| CUSTOM | 自定义 |

### 2.6 BudgetEntity（预算表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键，自增 |
| amount | Double | 预算金额 |
| month | String | 预算月份（格式：`"yyyy-MM"`，如 `"2026-04"`）|
| categoryId | Long? | 分类ID（**可为空**，`null` = 全局预算）|

## 三、数据流向

### 3.1 消费记录数据流

```
用户输入（AddExpenseScreen）
    ↓
AddExpenseViewModel.addExpense()
    ↓
ExpenseDao.insert() → ExpenseEntity
    ↓
DashboardViewModel 监听 ExpenseDao
    ↓
UI 更新（DashboardScreen）
```

### 3.2 首页数据流

```
App 启动 → startDestination = /dashboard
    ↓
DashboardScreen.loadDashboardData()
    ↓
DashboardViewModel 查询：
  - 本月支出（ExpenseDao.currentMonthTotal）
  - 本月收入（ExpenseDao.currentMonthIncome）
  - 近7日支出（ExpenseDao.recent7DaysTotal）
  - 每日分组消费（ExpenseDao.dailyExpenses）
  - 哇时刻预览（WeekStatsDao.wowPreview）
    ↓
LazyColumn 渲染
```

### 3.3 资产管理数据流

```
AssetManageScreen
    ↓
AssetManageViewModel 查询 AssetDao
    ↓
AssetEntity 列表展示
    ↓
用户操作（添加/编辑/删除）
    ↓
AssetDao 持久化
```

## 五、合规边界

- **不能做**：AI 直接替用户下单、投资建议模糊表述
- **能做**：信息聚合 + 客观呈现、风险提示、知识问答式助理
- **定位**：记账工具 + 消费洞察，信息服务而非投资顾问

## 六、规则引擎现状（代码层面）

> 参考：飞书 Wiki 第十章「智能消费决策」

### 6.1 现状：规则逻辑散布于 ViewModel 中

当前代码**没有独立的规则引擎模块**，规则逻辑以硬编码方式存在于各 ViewModel：

| 模块 | 文件 | 规则类型 |
|------|------|----------|
| 哇时刻 | `DashboardViewModel.kt` | 精准计算公式（已与 SPEC 一致）|
| 透视分析 | `InsightsViewModel.kt` | 阈值判断（硬编码阈值）|
| 成就/预算 | `AchievementsViewModel.kt` | 预算追踪规则 |

### 6.2 已实现的规则逻辑

#### 哇时刻规则
```kotlin
val avgLast4Weeks = recentStats.take(4).map { it.totalAmount }.average()
val savedAmount = avgLast4Weeks - recent7DaysTotalExpense
val isTriggered = savedAmount > 100 && savedAmount > avgLast4Weeks * 0.2
```

#### 透视分析规则（InsightsViewModel）
```kotlin
// 规模分析
if (total > 5000) → "规模较大，建议审查"

// 结构分析
if (topCategory.percentage > 0.4) → "存在优化空间"

// 频率分析
if (totalCount > 50) → "建议减少小额琐碎支出"
```

#### 预算追踪规则（AchievementsViewModel）
```kotlin
// 月度 vs 周度预算二选一
val activeType = if (monthlyBudget <= 0 && weeklyBudget > 0) "WEEKLY" else "MONTHLY"
```

### 6.3 目标形态（参考 Wiki 第十章）

```
规则引擎定位：计算层（精准）
    ↓
LLM：理解层（语义）
    ↓
用户：消费决策
```

**规则引擎职责**：
- 精准计算（超支金额、预算执行率、均值等）
- 条件触发（当某指标满足条件时触发洞察）
- 兜底保障（LLM 不可用时依然能提供基础建议）
- 历史数据聚合（同比/环比、趋势分析）

**LLM 职责**：
- 语义理解（备注解析）
- 公众信息整合（小红书等）
- 个性化表达（数字 → 温暖建议）
- 模糊判断（"值不值"）
