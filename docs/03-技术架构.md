## 一、技术架构

| 模块 | 方案 | 理由 |
|------|------|------|
| 记账入口 | 原生开发（安卓优先）| 本地性能+隐私控制 |
| 数据存储 | SQLite（本地）+ Room 抽象 | 隐私优先，钱迹同款方案 |
| 账单分析 | 本地AI推理（小模型）/ 外部 API | 不强依赖网络 |
| 数据来源 | V1：手动记账 | 降低门槛，合规 |
| UI 框架 | Jetpack Compose + Material3 | 现代声明式 UI |

### 技术栈总览

| 层级 | 技术选型 |
|------|----------|
| **前端** | Android Native（Kotlin + Jetpack Compose） |
| **本地数据库** | SQLite（通过 Room 抽象） |
| **AI 能力** | LLM API（MiniMax/DeepSeek/阿里百炼）|
| **网络** | Retrofit + OkHttp |
| **依赖注入** | Hilt |
| **异步处理** | Kotlin Coroutines + Flow |
| **架构模式** | MVVM + 简化 Clean Architecture（2层：data + presentation）|
| **后台任务** | WorkManager（哇时刻定时计算）|

## 五、合规边界

- **不能做**：AI 直接替用户下单、投资建议模糊表述
- **能做**：信息聚合 + 客观呈现、风险提示、知识问答式助理
- **定位**：记账工具 + 消费洞察，信息服务而非投资顾问

## 六、规则引擎现状（代码层面）

### 6.1 架构：独立 rules 模块

规则逻辑已从 ViewModel 抽取至独立 `rules/` 模块，通过 `RuleEngineTool` 对 LLM 暴露：

```
rules/
├── WowRule.kt         — 哇时刻触发判定
├── ScaleRule.kt       — 规模分析（阈值 ¥5000）
├── StructureRule.kt   — 结构偏向（占比 >40%）
├── FrequencyRule.kt   — 频率分析（笔数 >50）
├── Calculator.kt      — 通用计算（净资产/趋势点等）
└── Insight.kt        — 规则输出数据结构
```

### 6.2 规则逻辑

#### 哇时刻规则（WowRule）
```kotlin
fun isTriggered(savedAmount: Double, avgLast4Weeks: Double): Boolean {
    return savedAmount > 100.0 && savedAmount > avgLast4Weeks * 0.2
}
```

#### 规模规则（ScaleRule）
```kotlin
private const val SCALE_THRESHOLD = 5000.0
fun isLargeScale(total: Double): Boolean = total > SCALE_THRESHOLD
```

#### 结构规则（StructureRule）
```kotlin
private const val PERCENTAGE_THRESHOLD = 0.4f
fun isBiased(topPercentage: Float): Boolean = topPercentage > PERCENTAGE_THRESHOLD
```

#### 频率规则（FrequencyRule）
```kotlin
private const val COUNT_THRESHOLD = 50
fun isHighFrequency(totalCount: Int): Boolean = totalCount > COUNT_THRESHOLD
```

### 6.3 规则引擎职责分层

```
规则引擎（精准计算）
    ↓ 触发条件满足时返回 Insight
LLM（语义理解 + 表达）
    ↓
用户：消费决策
```

**规则引擎职责**：
- 精准计算（超支金额、预算执行率、均值等）
- 条件触发（当某指标满足条件时触发洞察）
- 兜底保障（LLM 不可用时依然能提供基础建议）
- 通过 `RuleEngineTool` 以 function calling 方式暴露给 LLM

**LLM 职责**：
- 语义理解（备注解析）
- 公众信息整合（小红书等）
- 个性化表达（数字 → 温暖建议）
- 模糊判断（"值不值"）

## 七、旺财 Micro Agent 系统

> 新增于 2026-04-03（commit: 52a490f / dfa3162 / 74c3c9b）

### 7.1 系统架构

```
LLMClient（DeepSeek-v3.2，流式/同步双模式）
    ↓
WangcaiAgent（Micro Agent，工具循环 + 进度反馈）
    ↓
ToolRegistry（工具注册表）
    ├── RuleEngineTool — 规则引擎读写（get_summary / scale_analysis / structure_analysis / frequency_analysis / record_user_explanation）
    ├── ContextTool    — 用户上下文读写（预算/目标/偏好/记忆）
    └── FileTool       — 本地文件读写
    ↓
AgentContext（内存 + JSON 文件持久化）
```

### 7.2 核心组件

| 组件 | 文件 | 职责 |
|------|------|------|
| **WangcaiAgent** | `agent/WangcaiAgent.kt` | Agent 编排，工具循环（最多 5 轮），流式响应，进度提示（`[PROGRESS: ...]`） |
| **LLMClient** | `agent/LLMClient.kt` | OpenAI 兼容接口，HTTP + SSE 流式读取，IO 线程 |
| **ToolRegistry** | `agent/ToolRegistry.kt` | 管理 3 个工具，提供 manifest（名称/描述/参数 schema） |
| **AgentContext** | `agent/AgentContext.kt` | JSON 上下文（预算/目标/记忆），内存缓存 + `wangcai_context.json` 文件持久化 |
| **AppInitializer** | `agent/AppInitializer.kt` | App 启动时 bootstrap（默认值填充/时间范围更新/分析计数） |
| **LLMModule** | `agent/di/LLMModule.kt` | Hilt DI，API Key 优先级（BuildConfig > EncryptedSharedPreferences），AES256-GCM 加密存储 |
| **RuleEngineTool** | `tool/RuleEngineTool.kt` | 暴露 rules 模块给 LLM，支持指定日期范围 |
| **ContextTool** | `tool/ContextTool.kt` | 暴露 AgentContext 读写给 LLM |
| **FileTool** | `tool/FileTool.kt` | 应用私有目录文件读写（LLM 可读写报告等） |

### 7.3 LLM 配置

| 参数 | 值 |
|------|-----|
| Provider | DeepSeek-v3.2 via Volcengine |
| Endpoint | `https://ark.cn-beijing.volces.com/api/coding/v3/chat/completions` |
| API Key 存储 | EncryptedSharedPreferences（AES256-GCM + MasterKey）|

### 7.4 系统提示词要点（WangcaiAgent）

```
1. 严禁使用 Markdown 表格，改用 Emoji + 列表
2. 先调用 get_summary 获取全量数据，再生成报告

4. 严格按照用户指定的日期范围描述，禁止惯性使用"最近30天"
```

## 八、本地记忆系统

> 新增于 2026-04-08（commit: 38bc692）

### 8.1 系统架构

```
对话消息
    ↓
ConversationStorage.addMessage()
    ├── 写入 MessageEntity（SQLite）
    ├── 异步索引到 FTS 表（关键词搜索）
    └── 异步生成向量（向量检索）
            ↓
    AI 回复后触发 Hook
            ↓
FactExtractor.extractFromAiResponse()
    ├── 获取最近 20 条对话上下文
    ├── 调用豆包 AI 提取关键事实
    └── 写入 ExtractedFactEntity + 向量索引
            ↓
HowToSpendViewModel.buildSystemContext()
    ├── MemoryRetriever.searchHybrid()（RRF 融合）
    ├── MemoryRefiner.buildMemorySummary()
    └── FactExtractor.buildFactsSummary()
            ↓
注入 AI Context
```

### 8.2 核心组件

| 组件 | 文件 | 职责 |
|------|------|------|
| **ConversationStorage** | `data/ConversationStorage.kt` | 会话/消息管理，触发向量索引和 fact 提取 |
| **MemoryRetriever** | `data/MemoryRetriever.kt` | FTS5/FTS4 全文搜索 + 向量相似度搜索 + RRF 融合检索 |
| **MemoryRefiner** | `data/MemoryRefiner.kt` | AI 记忆提炼，批量分析历史对话生成结构化画像 |
| **FactExtractor** | `data/FactExtractor.kt` | AI 关键信息提取，从对话中实时提取结构化事实 |
| **EmbeddingService** | `data/EmbeddingService.kt` | 文本向量化，调用远程 embedding API |

### 8.3 存储架构

#### 8.3.1 SQLite 表

| 表名 | 用途 |
|------|------|
| `messages` | 原始对话消息 |
| `sessions` | 会话元信息 |
| `memory` | AI 提炼的长期记忆（MemoryEntity）|
| `extracted_facts` | AI 实时提取的关键信息（ExtractedFactEntity）|

#### 8.3.2 向量存储

| 存储位置 | 表名 | 说明 |
|----------|------|------|
| `memory_fts.db` | `message_vectors` | 消息向量（768 维）|

#### 8.3.3 FTS 全文搜索

| 模式 | 表名 | 说明 |
|------|------|------|
| FTS5（优先）| `messages_fts` | 完整 FTS5 支持 |
| FTS4（降级）| `messages_fts` | 兼容性更好的 FTS4 |
| LIKE（兜底）| `messages_fallback` | 无 FTS 时的降级方案 |

**FTS 模式检测优先级**：`FTS5` → `FTS4` → `LIKE`

### 8.4 检索流程

```
用户查询
    ↓
FTS4 关键词检索（精准匹配）
    ↓
向量相似度搜索（语义匹配）
    ↓
RRF 融合（Reciprocal Rank Fusion）
    k = 60
    score = 1/(k + rank_fts) + 1/(k + rank_vector)
    ↓
返回融合后的 top-K 结果
```

### 8.5 关键信息提取（FactExtractor）

#### 提取触发时机

- AI 回复写入消息后自动触发
- 异步执行，不阻塞主流程
- 使用 `CoroutineScope(Dispatchers.IO)`

#### 提取 Prompt 示例

```
你是一个精准的信息提取助手。请从以下对话中提取用户透露的关键事实信息。

提取原则：
1. 只提取用户明确表达的事实，不要推测
2. key 命名格式：类型_具体内容（如 budget_clothes、income_salary）
3. value 使用 JSON 格式
4. summary 是人类可读的短句，20字以内
5. confidence 0-1
6. 只提取与财务相关的信息

输出格式：
{
  "facts": [
    {
      "key": "budget_monthly_clothes",
      "value": "{\"amount\": 1000, \"currency\": \"CNY\"}",
      "summary": "用户每月买衣服预算1000元",
      "confidence": 0.9
    }
  ]
}
```

#### 提取结果写入

```
ExtractedFactEntity
    ├── key = "budget_monthly_clothes"
    ├── value = "{\"amount\": 1000}"
    ├── summary = "用户每月买衣服预算1000元"  ← 向量嵌入对象
    ├── confidence = 0.9
    ├── source = "auto_extract"
    └── expiresAt = null  ← 永不过期
```

### 8.6 上下文注入

`HowToSpendViewModel.buildSystemContext()` 注入顺序：

1. **对话历史摘要**（当前 session）
2. **RRF 融合检索**（相关历史消息）
3. **财务画像**（资产/负债/预算/目标）
4. **MemoryRefiner 摘要**（AI 提炼的长期记忆）
5. **FactExtractor 摘要**（AI 提取的关键事实）
