# 💰 消费透视镜 · 技术架构

> 整理：玄枢·麒麟 | 最后更新：2026-03-31

---

## 一、技术架构

| 模块 | 方案 | 理由 |
|------|------|------|
| 记账入口 | 原生开发（安卓优先）| 本地性能+隐私控制 |
| 数据存储 | SQLite（本地）+ 备份到文件 | 隐私优先，钱迹同款方案 |
| 账单分析 | 本地AI推理（小模型）/ 外部 API | 不强依赖网络 |
| 数据来源 | V1：手动记账 | 降低门槛，合规 |

---

## 二、合规边界

- **不能做**：AI 直接替用户下单、投资建议模糊表述
- **能做**：信息聚合 + 客观呈现、风险提示、知识问答式助理
- **定位**：记账工具 + 消费洞察，信息服务而非投资顾问
