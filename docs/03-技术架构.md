## 一、技术架构

| 模块 | 方案 | 理由 |
|------|------|------|
| 记账入口 | 原生开发（安卓优先）| 本地性能+隐私控制 |
| 数据存储 | SQLite（本地）+ Room 抽象 | 隐私优先，钱迹同款方案 |
| 账单分析 | 本地AI推理（小模型）/ 外部 API | 不强依赖网络 |
| 数据来源 | V1：手动记账 | 降低门槛，合规 |
| UI 框架 | Jetpack Compose + Material3 | 现代声明式 UI |

### 技术栈总览

| 层级 | 技术选型 |
|------|----------|
| **前端** | Android Native（Kotlin + Jetpack Compose） |
| **本地数据库** | SQLite（通过 Room 抽象） |
| **AI 能力** | LLM API（MiniMax/DeepSeek/阿里百炼）|
| **网络** | Retrofit + OkHttp |
| **依赖注入** | Hilt |
| **异步处理** | Kotlin Coroutines + Flow |
| **架构模式** | MVVM + 简化 Clean Architecture（2层：data + presentation）|
| **后台任务** | WorkManager（哇时刻定时计算）|

## 二、Room 数据库架构（5张表）

### 2.1 数据库概览

```
AppDatabase (wealth_manager_db, version 4)
├── CategoryDao
│   └── CategoryEntity (分类表)
├── ExpenseDao
│   └── ExpenseEntity (消费记录表)
├── WeekStatsDao
│   └── WeekStatsEntity (周统计表)
├── AssetDao
│   └── AssetEntity (资产表)
└── BudgetDao
    └── BudgetEntity (预算表)
```

### 2.2 CategoryEntity（分类表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键，自增 |
| name | String | 分类名称 |
| icon | String | emoji 图标 |

**默认分类**：
- 🍗 餐饮
- 🛒 购物
- 🚇 交通
- 🎮 娱乐
- 🏠 居住
- 💊 医疗
- 📚 学习
- 📦 其他

### 2.3 ExpenseEntity（消费记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键，自增 |
| amount | Double | 金额 |
| categoryId | Long | 分类ID（外键）|
| note | String | 备注（最多50字）|
| date | Long | 日期时间戳 |
| type | Int | 类型（0=支出，1=收入）|

### 2.4 WeekStatsEntity（周统计表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键，自增 |
| weekStartDate | Long | 周开始日期时间戳 |
| totalSpend | Double | 周消费总额 |
| wowTriggered | Boolean | 是否触发哇时刻 |

### 2.5 AssetEntity（资产表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键，自增 |
| name | String | 资产名称 |
| amount | Double | 金额 |
| type | String | 类型（现金/银行卡/支付宝/微信/投资等）|

### 2.6 BudgetEntity（预算表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键，自增 |
| categoryId | Long | 分类ID（外键）|
| amount | Double | 预算金额 |
| month | Int | 预算月份（格式：YYYYMM）|

## 三、数据流向

### 3.1 消费记录数据流

```
用户输入（AddExpenseScreen）
    ↓
AddExpenseViewModel.addExpense()
    ↓
ExpenseDao.insert() → ExpenseEntity
    ↓
DashboardViewModel 监听 ExpenseDao
    ↓
UI 更新（DashboardScreen）
```

### 3.2 首页数据流

```
App 启动 → startDestination = /dashboard
    ↓
DashboardScreen.loadDashboardData()
    ↓
DashboardViewModel 查询：
  - 本月支出（ExpenseDao.currentMonthTotal）
  - 本月收入（ExpenseDao.currentMonthIncome）
  - 近7日支出（ExpenseDao.recent7DaysTotal）
  - 每日分组消费（ExpenseDao.dailyExpenses）
  - 哇时刻预览（WeekStatsDao.wowPreview）
    ↓
LazyColumn 渲染
```

### 3.3 资产管理数据流

```
AssetManageScreen
    ↓
AssetManageViewModel 查询 AssetDao
    ↓
AssetEntity 列表展示
    ↓
用户操作（添加/编辑/删除）
    ↓
AssetDao 持久化
```


## 五、合规边界

- **不能做**：AI 直接替用户下单、投资建议模糊表述
- **能做**：信息聚合 + 客观呈现、风险提示、知识问答式助理
- **定位**：记账工具 + 消费洞察，信息服务而非投资顾问
