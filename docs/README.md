# 📚 产品文档目录

本文档库包含消费透视镜的完整产品文档，与代码一起托管于 GitHub 仓库。

## 文档结构

```
docs/
├── README.md              ← 本文件
├── 01-产品背景与起源.md    ← 项目起源、市场分析、竞品分析
├── 02-产品设计.md          ← 核心定位、功能架构、产品机制
├── 03-技术架构.md          ← 技术路径、技术选型
├── 04-战略判断.md          ← 方向、风险、执行策略
├── 05-协作规范.md          ← gstack 虚拟团队工作流
└── 06-经验教训.md          ← 工作流沉淀、踩坑记录、方法论
```

## 文档与代码的关系

| 信息源 | 存储位置 | 说明 |
|--------|----------|------|
| 产品需求文档 | `docs/` | 唯一产品知识来源 |
| SPEC.md | 代码仓库根目录 | 技术规格说明书 |
| 飞书知识库 | ~~已废弃~~ | 文档已迁移至 GitHub |

## 版本管理

- 所有文档变更通过 Pull Request 审核后合并
- 文档版本与代码版本保持同步

## 参与贡献

产品文档采用与代码相同的协作流程：
1. 创建 Feature Branch
2. 提交文档修改
3. 通过 `/autoplan` 进行 CEO + 工程 + 设计评审
4. 合并到 main 分支
